# 1-D Marine Particle Coagulation Model

1-D depth-dependent model for marine particle coagulation, sinking,
and fragmentation. Built as a clean, step-by-step extension of the
0-D sectional model.

## Folder guide

- `src/` — OOP source classes used by all scripts
- `scripts/` — run scripts for each test and phase check
- `tests/` — unit checks for kernels, sinking speeds, transport, and conservation
- `output/` — figures, logs, and tables (generated by scripts, not in git)

## Source classes (src/)

| Class | Purpose |
|---|---|
| `SimulationConfig` | all model parameters |
| `ColumnGrid` | depth grid: midpoints, faces, spacing |
| `DepthProfile` | T(z), S(z), rho(z), Kz(z), nu(z) with `DepthProfile.typical()` constructor |
| `ColumnTransport` | upwind advection + flux-form diffusion, one time step |
| `ColumnRHS` | full step: transport then depth-scaled process rates per layer |
| `ColumnSimulation` | top-level driver: grid, profile, RHS, time loop |
| `BetaAssembler` | builds Brownian, shear, and DS collision kernels |
| `CoagulationRHS` | sectional coagulation gain/loss |
| `SettlingVelocityService` | sinking laws: kriest_8, kriest_9, siegel_2025, current |
| `Disaggregation` | legacy fragmentation |
| `DisaggregationOperatorSplit` | operator-split fragmentation via d_max(eps) |

## Current settings (May 2026)

- Sinking law: `kriest_8`
- DS kernel mode: `sinking_law`
- Column depth: 1000 m, 20 depth cells, 5 size sections
- Time step: 1 day with `proc_substeps = 20`
- Scheme: upwind
- Fragmentation: operator-split preferred; legacy safe only at c3 ≤ 0.002
